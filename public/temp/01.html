Serving static files with Express.js

AKA: The WHY THE FUCK ARE MY IMAGES NOT LOADING? problem.

By default, when you run a web app with Express, it will refuse requests to serve any resources in the project folder. I imagine that this prevents, for example, someone from doing a request like GET db/schema to get the files that reveal our database structure and give them some critical information about how to attack our site or obtain sensitive information, or to GET server.js to learn more about the back-end logic of our application and how to side-step it or learn any weak points.

So if you want to make Express serve your static files (such as our images), you need to tell Express to serve those files! It’s totally fine if you didn’t recognize this, but we have covered it in class, so it would be good to review the Express documentation page that talks about this. It’s not long, and you likely won’t have trouble understanding it. If you do, let me know:

https://expressjs.com/en/starter/static-files.html

We also encountered this issue once earlier this week — remember when I was having trouble getting the banner photo to render on the homepage and asked you both for some help?

I had placed the file in public/img but it was still giving a 404 error. It turned out that Express didn’t have permission to serve the files, and to fix this we added this to server.js:

app.use(express.static("public"));

And once we added this, the image rendered properly, and we were able to access it in our browser by going to http://localhost:8080/img/open-kitchen-banner.jpg — note we didn’t have to include public in the path because Express treats public/ as a root and begins searching from there.

So the solution to getting the images to render on the storefront was to add a single line:

app.use(express.static("img"));

Now Express will treat the img/ folder as another root from which to begin a search for a static resource, so the images will now render on the storefront, and the images can all be accessed directly in the browser with the URL structure http://localhost:8080/food_01.jpg (note the img/ is not needed) ———>

Then, to prove all the images would render to the page, I just used a for loop with some EJS:

<% for( let i = 0; i < menuItems.length; i++ ) { %>
  <img src="<%= menuItems[i].photo.slice(4) %>" width="350px">
  <p>
    <%= menuItems[i].photo %>
  </p>
<% } %>
