Fixing the database connection error

AKA: The WHY THE FUCK DOES THE DATABASE GIVE A CONNECTION ERROR BUT STILL RETURN DATA FROM DATABASE QUERIES, AND WHY AREN'T WE ALL GETTING THE SAME FUCKING ERROR? problem.

You may remember that Pat and I have been getting a database connection error on server startup since Julia made some changes to the database code:

My first step was to try to understand where the issue is happening. First I tried adding some console.log() in server.js on the code that is triggered for the GET / route (around line 70 in server.js), which revealed that the error was happening before the request. In hindsight, this should have been obvious, because the error fires off only a single time, on server start-up, and not related to any routing requests. This was a big tip!

So I looked for the thing in server.js that has to do with the database and happens only once on startup, and at no other time during the operation of our program. This brought me to our database connection code:

const dbParams = require("./lib/db.js");
const db = new Pool(dbParams);
db.connect();

From reading this code, I would expect dbParams to be an object containing the credentials needed for database login, but given the error I keep getting, maybe there is an issue there. So I tried logging dbParams with console.log(dbParams); right before the database connection is established, and I got this:

Things to note:

  1. The dbParams object is being logged before the error happens, which proves that I have zeroed in on where the problem is happening.
  1. dbParams is NOT simply an object containing the database login credentials, as we expected. In reality, dbParams is an object containing EVERYTHING that was exported in lib/db.js and consequently imported in server.js— the database credentials as well as every other function written in that file — because look again at how it was imported:
const dbParams = require("./lib/db.js");
  1. Julia has probably realized by now that this is the exact same mistake that tripped her up at an earlier part in this project.

The fix...

...is deceptively simple once you see the problem. Just use property value shorthand by adding curly brackets in order to extract only the database credentials:

const { dbParams } = require("./lib/db.js");

Now when I log to console, I get this —>

dbParams only contains the credentials, and no database login error on server startup!

That’s the best solution if you only need the credentials. But if you to have all the other functions on hand, then you can import like this:

const dbHelpers = require("./lib/db.js");

And then you can select just the database credentials like this:

const db = new Pool(dbHelpers.dbParams);
